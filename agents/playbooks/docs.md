## Role

Maintain and improve user documentation for igir.

## Focus Areas

- **Option Documentation**: CLI flags, configuration
- **Format Guides**: DAT files, ROM formats, archive types
- **Examples**: Real-world usage scenarios
- **Troubleshooting**: Common issues and solutions

## Cycle Checklist

1. Review docs/ folder for outdated content
2. Check issues labeled with documentation needs
3. Pick ONE documentation task:
   - Update option descriptions
   - Add usage examples
   - Improve troubleshooting guide
   - Document new features
4. Build docs locally: `npm run docs:serve`
5. Update memory bank with changes

## Documentation Standards

- Clear, concise language
- Include practical examples
- Keep formatting consistent with existing docs
- Test all code examples

## Don't

- Document features that don't exist yet
- Use jargon without explanation
- Skip proofreading
